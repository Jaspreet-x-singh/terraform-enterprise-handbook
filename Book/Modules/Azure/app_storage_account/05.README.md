# Storage Account Module

This Terraform module deploys an Azure Storage Account following AzureRM provider best practices.  
It supports all major optional features including network rules, CORS, static website hosting, hierarchical namespace (ADLS Gen2), Azure Files authentication, and advanced security settings.

The module is designed for use inside a multi‑module Terraform architecture, where each root module consumes reusable child modules.

---

## 📦 Features

- Creates a Storage Account with secure defaults
- Supports all optional Azure Storage features:
  - HTTPS‑only traffic
  - Cross‑tenant replication
  - Public network access control
  - Shared access key enable/disable
  - Hierarchical namespace (ADLS Gen2)
  - NFSv3 support
  - Large file shares
  - Local user (SFTP) support
  - Infrastructure encryption
  - Network rules (IP, VNet, bypass)
  - CORS rules
  - Static website hosting
  - Azure Files authentication
  - Retention policy
- Fully validated inputs
- Optional arguments applied only when set
- Clean, minimal outputs

## 📁 Module Structure

```text
modules/storage_account/
├── main.tf
├── variables.tf
├── outputs.tf
└── README.md

module "storage" {
  source = "../modules/storage_account"

  name                     = "mystorage123"
  resource_group_name      = var.resource_group_name
  location                 = var.location
  account_tier             = "Standard"
  account_replication_type = "LRS"

  https_traffic_only_enabled = true
  shared_access_key_enabled  = false

  network_rules = {
    default_action = "Deny"
    ip_rules       = ["1.2.3.4"]
  }

  cors_rule = [
    {
      allowed_headers    = ["*"]
      allowed_methods    = ["GET", "POST"]
      allowed_origins    = ["*"]
      exposed_headers    = ["*"]
      max_age_in_seconds = 3600
    }
  ]

  static_website = {
    index_document     = "index.html"
    error_404_document = "404.html"
  }

  tags = {
    environment = "prod"
    owner       = "platform-team"
  }
}
```

## 🧩 Inputs

| Name | Type | Description | Required |
|------|------|-------------|----------|
| `name` | string | Globally unique Storage Account name. | Yes |
| `resource_group_name` | string | Resource Group name. | Yes |
| `location` | string | Azure region. | Yes |
| `account_tier` | string | Standard or Premium. | Yes |
| `account_replication_type` | string | LRS, GRS, RAGRS, ZRS. | Yes |
| `account_kind` | string | StorageV2, Storage, BlobStorage, etc. Default: StorageV2. | No |
| `access_tier` | string | Hot or Cool. Default: Hot. | No |
| `min_tls_version` | string | TLS1_0, TLS1_1, TLS1_2. Default: TLS1_2. | No |
| `tags` | map(string) | Resource tags. | No |

## 🔒 Security Best Practices

- HTTPS‑only should be enabled unless explicitly required otherwise.
- Shared access keys should be disabled when using Azure AD authentication.
- Public network access should be disabled for private‑only deployments.
- ADLS Gen2 (`is_hns_enabled`) cannot be changed after creation.
- NFSv3 requires hierarchical namespace enabled.
- Infrastructure encryption should be enabled for highly sensitive workloads.
- Avoid enabling `allow_nested_items_to_be_public` unless absolutely necessary.
- Use network rules to restrict access to trusted IPs or VNets.
