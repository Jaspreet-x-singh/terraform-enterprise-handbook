# Monitoring Module  

### Log Analytics Workspace + Application Insights + Diagnostic Settings

This module deploys:

- Azure Log Analytics Workspace  
- Azure Application Insights (linked to LAW)  
- Azure Monitor Diagnostic Settings (optional)

All optional Application Insights arguments are applied **only when non-null**, ensuring clean Terraform plans.

---

## 🧩 Inputs

### Log Analytics Workspace

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `workspace_name` | string | n/a | Name of the Log Analytics Workspace. |
| `location` | string | n/a | Azure region. |
| `resource_group_name` | string | n/a | Resource group name. |
| `workspace_sku` | string | `"PerGB2018"` | Workspace SKU. |
| `workspace_retention_days` | number | `30` | Retention period (30–730 days). |

---

### Application Insights (Core)

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `app_insights_name` | string | n/a | Application Insights name. |
| `app_insights_application_type` | string | `"web"` | Application type. |

---

### Application Insights (Optional Arguments)

All optional arguments default to `null` and are **not applied unless explicitly set**.

| Name | Type | Default |
|------|------|---------|
| `disable_ip_masking` | bool | null |
| `local_authentication_disabled` | bool | null |
| `internet_ingestion_enabled` | bool | null |
| `internet_query_enabled` | bool | null |
| `force_customer_storage_for_profiler` | bool | null |
| `sampling_percentage` | number | null |
| `daily_data_cap_in_gb` | number | null |
| `daily_data_cap_notifications_disabled` | bool | null |
| `retention_in_days` | number | null |
| `tags` | map(string) | `{}` |

---

### Diagnostic Settings

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `enabled` | bool | `true` | Global module enable switch. |
| `diagnostic_setting_enable` | bool | `false` | Enables diagnostic settings. |
| `storage_account_id` | string | null | Storage account for logs. |
| `eventhub_name` | string | null | Event Hub name. |
| `eventhub_authorization_rule_id` | string | null | Event Hub auth rule. |
| `log_analytics_workspace_id` | string | null | LAW ID for diagnostic settings. |
| `log_analytics_destination_type` | string | `"Dedicated"` | Destination type. |
| `category` | string | `"AppTraces"` | Log category. |

---

## 📤 Outputs

| Output | Description |
|--------|-------------|
| `log_analytics_workspace_id` | ID of the Log Analytics Workspace. |
| `application_insights_id` | ID of the Application Insights instance. |
| `application_insights_connection_string` | Connection string for Application Insights. |
| `diagnostic_setting_id` | ID of the diagnostic setting (if enabled). |

---

## 📘 Diagnostic Settings Behaviour

Diagnostic settings are created only when:

enabled = true
diagnostic_setting_enable = true

The module configures:

- `enabled_metric { category = "AllMetrics" }`
- `enabled_log { category = var.category, category_group = "AllLogs" }`

Supported destinations:

- Log Analytics Workspace  
- Storage Account  
- Event Hub  

Any destination set to `null` is **not applied**.
