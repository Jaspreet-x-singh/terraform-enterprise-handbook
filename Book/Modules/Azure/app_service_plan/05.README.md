# Azure App Service Plan Module

This module creates an Azure App Service Plan using the `azurerm_service_plan` resource.  
It includes full validation, conditional logic, and automated testing support to ensure only valid App Service Plan configurations are deployed.

---

## Inputs

| Name                       | Type                                                                 | Default | Description |
|----------------------------|----------------------------------------------------------------------|---------|-------------|
| name                       | string                                                               | n/a     | Name of the App Service Plan |
| resource_group_name        | string                                                               | n/a     | Resource group name |
| location                   | string                                                               | n/a     | Azure region |
| os_type                    | string                                                               | n/a     | OS type: `Linux` or `Windows` |
| sku_name                   | string                                                               | n/a     | SKU name (e.g., `B1`, `S1`, `P1v3`) |
| app_service_environment_id | string                                                               | null    | App Service Environment ID |
| worker_count               | number                                                               | null    | Number of workers (only valid for Premium, PremiumV2, PremiumV3, Isolated, IsolatedV2, Elastic Premium) |
| per_site_scaling           | bool                                                                 | false   | Enable per-site scaling (only valid for Standard and Premium tiers) |
| zone_balancing_enabled     | bool                                                                 | false   | Enable zone redundancy (only valid for PremiumV3 and IsolatedV2) |
| tags                       | map(string)                                                          | {}      | Resource tags |
| timeouts                   | object({ create, read, update, delete = optional(string) })         | {}      | Timeouts block |

---

## Validation Rules

### OS Type
Allowed values:
- `Linux`
- `Windows`

### SKU Name
Supported SKUs:

- **Free:** `F1`  
- **Shared:** `D1`  
- **Basic:** `B1`, `B2`, `B3`  
- **Standard:** `S1`, `S2`, `S3`  
- **Premium:** `P1`, `P2`, `P3`  
- **PremiumV2:** `P1v2`, `P2v2`, `P3v2`  
- **PremiumV3:** `P1v3`, `P2v3`, `P3v3`  
- **Isolated:** `I1`, `I2`, `I3`  
- **IsolatedV2:** `I1v2`, `I2v2`, `I3v2`  
- **Elastic Premium:** `EP1`, `EP2`, `EP3`  
- **Consumption:** `Y1`  

### Worker Count
Allowed only for:
- Premium / PremiumV2 / PremiumV3  
- Isolated / IsolatedV2  
- Elastic Premium  

### Per-Site Scaling
Allowed only for:
- Standard  
- Premium / PremiumV2 / PremiumV3  

### Zone Balancing
Allowed only for:
- PremiumV3  
- IsolatedV2  

---

## Conditional Logic in the Module

The module automatically prevents invalid provider arguments:

- `worker_count` is set only when the SKU supports it  
- `per_site_scaling` is enabled only for valid SKUs  
- `zone_balancing_enabled` is enabled only for valid SKUs  

Invalid combinations are safely ignored or set to defaults.

## Outputs

| Name      | Description |
|-----------|-------------|
| id        | ID of the App Service Plan |
| name      | Name of the App Service Plan |
| location  | Location of the App Service Plan |
| sku_name  | SKU of the App Service Plan |
| os_type   | OS type of the App Service Plan |

## Example Usage

```hcl
module "app_service_plan" {
  source = "github.com/your-org/terraform-azure-modules//app_service_plan"

  name                = "asp-demo-prod"
  resource_group_name = azurerm_resource_group.rg.name
  location            = azurerm_resource_group.rg.location
  os_type             = "Linux"
  sku_name            = "P1v3"

  worker_count           = 2
  per_site_scaling       = false
  zone_balancing_enabled = true

  tags = {
    environment = "prod"
    workload    = "web"
  }

  timeouts = {
    create = "30m"
    update = "30m"
  }
}
