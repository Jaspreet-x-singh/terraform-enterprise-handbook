# Contributing Guidelines

Thank you for your interest in contributing to  
**Terraform: The Complete Practical Guide — Extended Enterprise Edition**.

Contributions of all kinds are welcome, including:

- Fixing typos or formatting  
- Improving explanations  
- Adding diagrams  
- Enhancing examples  
- Suggesting new patterns or best practices  
- Reporting issues  

---

## 🧱 How to Contribute

1. Fork the repository  
2. Create a new branch  
3. Make your changes  
4. Submit a pull request with a clear description  

---

## 📘 Writing Style

Please follow these guidelines:

- Use clear, concise technical language  
- Prefer examples over theory  
- Use Markdown formatting consistently  
- Keep diagrams in `/book/diagrams`  
- Keep Terraform examples in `/book/examples`  

---

## 🧪 Validating Your Contribution

Before submitting:

- Run spellcheck  
- Validate Markdown formatting  
- Ensure links are correct  
- Keep chapter numbering consistent  

---

## 🙏 Thank You

Your contributions help improve the quality and reach of this book.  
Thank you for helping make Terraform knowledge accessible to everyone.
